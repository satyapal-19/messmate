Payment + Verification Flow
Student books meal
       ↓
System generates MM-XXXX code (expires 4h, single-use)
       ↓
Student shows code/QR at counter
       ↓
Vendor enters code in app → validates:
  • Code exists and unexpired
  • Student has active booking for this slot/date
  • Code not already used (anti-fraud)
       ↓
Vendor confirms cash received
       ↓
Booking status → VERIFIED, timestamp logged
       ↓
Student can now rate the meal
Anti-Fraud Protections

Codes are single-use and expire in 4 hours
One booking per student per meal slot per day
Rate limiting: max 3 code regenerations/day
Subscription skip abuse detection (>6 skips flags account)
Vendor verification requires biometric/PIN on device
